# Basim trend-based carousel preview workflow

Use this when Basim asks for a carousel example whose slide content should come from current trends/news.

## Durable lesson

Basim challenged dummy carousel copy and asked where the slide content came from. For trend-based carousel examples, the content should not be generic evergreen copy. It should be grounded in current web-search signals and disclosed briefly in the preview response.

## Preferred sequence

1. Ground the date/time with `TZ=Asia/Jakarta date`.
2. Run multiple web searches for today's Indonesian trends/news, with at least one query constrained to today's date and at least one safe/business-relevant vertical such as technology, digital behavior, creator economy, UMKM, finance, or consumer behavior.
3. Prefer sources published today. If the best source is older but still actively discussed, label that explicitly and do not call it the newest.
4. Avoid sensitive/crime/accident topics for auto-preview unless Basim explicitly asks; use safer business/consumer/digital angles when possible.
5. Convert the source into a carousel angle:
   - source fact / trend signal,
   - human/business implication,
   - hidden system,
   - practical takeaway.
6. Generate with the normal SenangCarousels visual template unless Basim explicitly requests visual experimentation.
7. Send contact sheet and ZIP to the chat first. Do not schedule to Repliz until Basim explicitly approves.

## Preview response should include

- Media contact sheet.
- ZIP path.
- Verification: HTTP status, slide count, dimensions, ZIP validity.
- Source/date used.
- One-line angle rationale.
- Explicit note: not scheduled yet.

## Pitfall

Do not say a carousel is based on viral radar/current trends if the slide copy was manually invented from generic Basim themes. If a placeholder is used for technical testing, label it as dummy/manual.
