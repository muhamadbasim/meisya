# Basim approved avatar+hologram square carousel style

Use this reference when Basim wants carousel slides that match the approved June 2026 visual direction.

## Core visual DNA

- Format: square `1080x1080` by default.
- Background: plain white.
- Composition: text-first, upper-middle, lots of negative space.
- Text: exactly 3 centered lines.
- Typography: modern sans-serif, lowercase.
- Emphasis: one bold keyword per line, usually at the end of the line.
- Accent: Basim 3D avatar/commander at bottom-right.
- Tech accent: blue hologram/dashboard panels behind or around the avatar only.
- Small detail: subtle grey chevron/caret near bottom center.
- Do not use borders, serif fonts, dense paragraph blocks, infographic clutter, or full-background tech overlays.

## Approved copy pattern

Preferred line structure:

```text
[regular] [bold keyword]
[regular] [bold keyword]
[regular] [bold keyword]
```

Examples from the approved reference set:

- `ai **ramai** / bisnis **bingung** / mulai dari **mana**`
- `cek **manual** / sebelum naik **publik** / biar tetap **manusiawi**`
- `pakai **ai** / untuk draf **awal** / bukan suara **akhir**`
- `beri **konteks** / sebelum minta **jawaban** / siapa butuh **apa**`
- `konten **cepat** / bisa tetap **dingin** / kalau tanpa **rasa**`

## Workflow lesson from this session

If Basim says a draft is "tidak seperti ini", the visual reference wins immediately.

Common miss to avoid:
- border/frame layouts
- serif title fonts
- long quote-card paragraphs
- missing avatar/hologram accent
- 4:5 mockups when the approved style is square
