# Approved Basim avatar+hologram carousel template

Use this reference when Basim wants the currently-approved social carousel look.

## Visual DNA

- Format: square `1080x1080` by default.
- Background: plain white.
- Layout: text-first with very large negative space.
- Text position: centered horizontally in the upper-middle area.
- Copy structure: exactly **3 short lines**.
- Typography: lowercase modern sans-serif, black text.
- Emphasis: **one bold keyword per line**, usually the last word.
- Avatar: small 3D Basim/commander character at bottom-right.
- Accent: blue hologram/dashboard behind the avatar only, not full-canvas.
- Optional subtle chevron/caret near bottom center.
- Avoid: borders, serif fonts, long paragraphs, infographic density, busy backgrounds.

## Copy mechanics

Preferred formula:

```text
[regular] **keyword**
[regular] **keyword**
[regular] **keyword**
```

Each slide should usually communicate **one punchy insight**, not a dense explanation.

## Workflow lesson from this session

If Basim says a draft is "not like this" and provides a reference image:

1. Treat the image as the authoritative template.
2. Compare your draft against the reference on format, text density, font family, bold placement, avatar placement, and hologram usage.
3. Regenerate a small test set/contact sheet first.
4. Wait for Basim's visual approval before using the style in production jobs.
5. Once Basim says the draft is "sudah sesuai", treat that as the locked current template for future carousels.
