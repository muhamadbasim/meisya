# Beginner AI content notes for Basim Threads

Use this reference when the user wants simple AI tips/tricks content for beginners.

## Research-backed basics

Across mainstream prompting guidance from OpenAI, Anthropic, Google/Gemini, MIT Sloan, and beginner AI guides, the repeated practical points are:

- Clear instructions beat vague requests.
- Add context: audience, goal, constraints, and background.
- Specify output format: bullet list, caption options, table, script, checklist, etc.
- Specify tone: casual, professional, funny, direct, beginner-friendly.
- Give examples when style matters.
- Ask the AI to clarify if important information is missing.
- Verify outputs when they involve facts, prices, law, health, current events, or specific numbers.

## Beginner-friendly formula

Avoid jargon like “prompt engineering” unless needed. Teach it as:

```text
Kasih tahu AI:
- buat siapa
- tujuannya apa
- konteksnya apa
- gaya bahasanya gimana
- hasilnya mau bentuk apa
```

A compact formula if a formula is useful:

```text
Peran + tugas + konteks + format + gaya
```

Example:

```text
Anggap kamu admin toko online.
Buat 5 caption jualan sepatu wanita.
Targetnya ibu muda usia 25–35.
Gaya bahasanya santai.
Formatnya bullet pendek.
```

## Basim-style Threads patterns

### Casual curhat

```text
Kadang orang bilang AI-nya nggak pinter.

Padahal prompt-nya cuma:
“buat konten dong”

Ya AI-nya juga nebak-nebak 😅

Coba kasih konteks dikit:
targetnya siapa,
jualannya apa,
mau gaya santai atau serius,
outputnya mau berapa poin.

AI itu makin enak dipakai kalau kita makin jelas ngomongnya.
```

### Two-sentence variant

```text
Kalau baru belajar AI, jangan cuma nulis “buat caption”, karena AI akhirnya nebak-nebak sendiri.
Coba kasih konteks: buat siapa, tujuannya apa, gaya bahasanya gimana, dan hasilnya mau bentuk apa.
```

### Analogy variant

```text
Pakai AI itu mirip nyuruh orang bantu kerja.

Kalau instruksinya cuma “tolong bikinin”, hasilnya bisa ke mana-mana.

Tapi kalau dikasih konteks, contoh, dan format yang jelas, hasilnya biasanya langsung lebih dekat.
```

## Pitfalls

- Do not over-teach with a formal classroom tone unless the user asks.
- Do not turn every post into a listicle; the user often wants chatty/spontaneous copy.
- Do not claim AI is always accurate. Include verification reminders when discussing facts.
- Keep examples local and ordinary: caption jualan, chat customer, ringkas meeting, ide konten, admin toko online.
